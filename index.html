OK, that's nice, what is it really?

From the site:
D3.js is a JavaScript library for manipulating documents based on data. D3 helps you bring data to life using HTML, SVG and CSS. D3's emphasis on web standards gives you the full capabilities of modern browsers without tying yourself to a proprietary framework, combining powerful visualization components and a data-driven approach to DOM manipulation.

In other words:
D3.js is an api wrapper around SVG elements which makes it easy to manipulate multiple data points.

What is SVG?

From Wikipedia:

Scalable Vector Graphics (SVG) is an XML-based vector image format for two-dimensional graphics with support for interactivity and animation. The SVG specification is an open standard developed by the World Wide Web Consortium (W3C) since 1999.

SVG images and their behaviors are defined in XML text files. This means that they can be searched, indexed, scripted, and compressed. As XML files, SVG images can be created and edited with any text editor, but are more often created with drawing software.

Quick D3.js primer
  • select/selectAll: Selects HTML elements. The main difference between this and other libraries is that D3's select can also select elements which do not yet exist.
    Example Code: 
    > d3.selectAll('.example_select li')
 [ Array(3) ]
    Example Data:
    • 5
    • 7
    • 9
  • data.enter: Binds data to a selected element.
    Note that most functions that update data take either the specific data you want to update: "1", or alternately a function that receives the data point and the index: function(d, i)
    Example Code: 
    > d3.selectAll('.example_enter li')
    .data([1,2,3])
    .enter()
    .append('li')
    .text(function(d) { return d; })
 [ Array(3) ]
    Example Data:
    • 1
    • 2
    • 3
  • Scales: Functions that scale the data:
    • Linear - maps one range of data to another. 
      > var scale = d3.scale.linear().domain([0, 10]).range([0,100])
> scale(0)
 0
> scale(1)
 10
> scale(10)
 100
    • Ordinal - maps a range of data to set values 
      > var scale = d3.scale.ordinal().domain([1,2,3]).range(['one', 'two', 'three'])
> scale(1)
 'one'
> scale(2)
 'two'
> scale(3)
 'three'
Why use D3.js?

  • You can create any kind of chart you want.

    This is really the selling point, most other charting libraries provide a set of predefined charts that you can use and hopefully provide the options that you want. With D3.js you can generate anything you can conceive of.

  • While it won't replace things like jQuery and Angular the API is generally logical and consistent.
  • It provides easy ways to generate the more complex SVG elements such as paths.
Why not use D3.js?
  • IE8 does not support the SVG element.

  • You can create anything, but you also have to.

    While there are utilities for say figuring out how to split up the slices of a pie chart, there is no default legend element.

    There may be ways to modularize some of that functionality, but few of the examples even try. So it does not seem to be a priority of the library.

  • In some ways it tries to do too much like API calls for things such as JSON data which aren't strictly necessary for charting. It is interesting that you can do everything you need in d3.js, but it seems like some of that functionality should be passed on to another library that is meant for it rather than reinventing the wheel.
